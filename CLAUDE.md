## Project

**heavy-slug** is a Zig 0.16.0-dev.3091+ GPU text rendering library. It uses the Slug algorithm (Eric Lengyel) for exact quadratic Bezier coverage via Vulkan mesh shaders. The library module (`src/root.zig`) is importable by external packages; `src/main.zig` is a demo executable.

## Commands

```bash
# Format
zig fmt src/

# Build library + executable
zig build

# Run executable
zig build run
zig build run -- arg1 arg2

# Run all tests (silent on success)
zig build test

# Build with optimization
zig build -Doptimize=ReleaseFast   # or ReleaseSafe, ReleaseSmall
zig build -Dtarget=<target>
```

## Key Patterns

**Test discovery:** `src/root.zig` must `_ = @import(...)` each module in its `test` block for nested tests to run. `zig build test` produces no output on success.

**C library builds:** `buildFreetype()` and `buildHarfbuzz()` in `build.zig` return `*std.Build.Step.Compile`. `@cImport` wrappers need `mod.addIncludePath()` ŌĆö `linkLibrary` alone does not propagate headers.

**Vulkan binding generation:** Manual pattern ŌĆö get the `vulkan-zig-generator` artifact, run it with `registry/vk.xml`, create a module from the output:
```zig
const registry = b.dependency("vulkan_headers", .{}).path("registry/vk.xml");
const vk_gen = b.dependency("vulkan", .{}).artifact("vulkan-zig-generator");
const vk_generate_cmd = b.addRunArtifact(vk_gen);
vk_generate_cmd.addFileArg(registry);
const vulkan_zig = b.addModule("vulkan-zig", .{
    .root_source_file = vk_generate_cmd.addOutputFileArg("vk.zig"),
});
```

**Vulkan filtered dispatch:** `vk.DeviceWrapper(.{...})` does NOT exist. Define a struct with `?vk.PfnXxx = null` fields, pass to `vk.DeviceWrapperWithCustomDispatch(YourStruct)`. See `src/gpu/context.zig` for the `HeavySlugDispatch` pattern.

**Cross-module @cImport interop:** `ft.zig` and `hb.zig` each have their own `@cImport` blocks including FreeType headers ŌĆö Zig treats them as distinct type namespaces. Passing `FT_Face` across modules uses `*anyopaque` as the bridge: `ft.Face.rawHandle()` returns `@ptrCast(self.handle.?)`, and `hb.Font.createFromFtFace` reconstructs with `@ptrCast(@alignCast(ptr))`.

**Test fonts:** Font tests use `assets/Inter-Regular.otf` (relative to the project root, where `zig build test` runs). Do not hardcode system font paths.

**Font pipeline:** `glyph.FontContext` owns FT_Face + hb_font + reusable hb_gpu_draw. Call `shapeText(text, null, null)` for auto-detected direction/script (calls `hb_buffer_guess_segment_properties`), or pass explicit `hb.Direction`/`hb.Script` values. `encodeGlyph(glyph_id)` runs the drawGlyphŌåÆencodeŌåÆgetExtentsŌåÆreset cycle and returns an `EncodedGlyph` (caller owns the blob).

**HarfBuzz GPU draw cycle:** `GpuDraw.drawGlyph` ŌåÆ `GpuDraw.encode` (returns `Blob`) ŌåÆ `GpuDraw.getExtents` ŌåÆ `GpuDraw.reset`. Always call reset after encode (or on encode failure via errdefer) to avoid dirty state on the next glyph.

**Glyph cache:** `cache.GlyphCache` maps `(font_id, glyph_id)` to descriptor slot + pool allocation. Hot tier holds ASCII/promoted glyphs (evicted only on font unload). Cold tier uses LRU eviction. Glyphs used for `promote_frames` consecutive frames are promoted coldŌåÆhot. `pool.PoolAllocator` manages byte-range sub-allocations within the glyph blob VkBuffer (bump + free-list, configurable alignment for `minStorageBufferOffsetAlignment`).

**Motor math:** `src/math/pga.zig` ŌĆö storage is `[4]f32` extern struct (GPU ABI). Arithmetic uses `@Vector(4,f32)` + `@splat` + `@shuffle` internally. `Motor.apply` uses `@Vector(2,f32)`.

## Zig 0.16.0-dev Gotchas

- `b.addLibrary(.{ .linkage = .static })` ŌĆö NOT `b.addStaticLibrary()` (removed in 0.16)
- `@cImport` needs explicit `mod.addIncludePath()` ŌĆö `linkLibrary` alone doesn't propagate headers
- `zig build test` is silent on success (exit 0, no output)
