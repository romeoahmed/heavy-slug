## Project

**heavy-slug** is a Zig 0.16.0-dev.3091+ GPU text rendering library. It uses the Slug algorithm (Eric Lengyel) for exact quadratic Bezier coverage via Vulkan mesh shaders. The library module (`src/root.zig`) is importable by external packages; `src/main.zig` is a demo executable.

## Commands

```bash
# Format
zig fmt src/

# Build library + executable
zig build

# Run executable
zig build run
zig build run -- arg1 arg2

# Run all tests (silent on success)
zig build test

# Build with optimization
zig build -Doptimize=ReleaseFast   # or ReleaseSafe, ReleaseSmall
zig build -Dtarget=<target>
```

## Architecture

```
src/
  root.zig          — public API re-exports (ft, hb, pga, gpu_context, ...)
  main.zig          — demo executable
  font/
    ft.zig          — FreeType @cImport wrapper
    hb.zig          — HarfBuzz @cImport wrapper + hb_gpu_draw wrappers
  gpu/
    context.zig     — DeviceDispatch (HeavySlugDispatch filtered wrapper)
    descriptors.zig — DescriptorTable: storage buffer array[65536]       [Plan 5]
    pipeline.zig    — Mesh+fragment pipeline, push constant layout        [Plan 7]
    renderer.zig    — TextRenderer public API                             [Plan 9]
    layout.zig      — AUTO-GENERATED by tools/layout_gen                 [Plan 10]
  math/
    pga.zig         — Cl(2,0,1) Motor/Point, @Vector(4,f32) SIMD internals
shaders/
  pga.slang         — Motor struct (mirrors Zig)                          [Plan 6]
  slug_task.slang   — Task shader: frustum cull, payload emit             [Plan 6]
  slug_mesh.slang   — Mesh shader: SlugDilate + quad emit                 [Plan 6]
  slug_fragment.slang — Fragment shader: Slug band/coverage               [Plan 6]
tools/
  layout_gen.zig    — codegen: layout.json → src/gpu/layout.zig          [Plan 10]
```

Items marked `[Plan N]` are not yet implemented.

## Key Patterns

**Test discovery:** `src/root.zig` must `_ = @import(...)` each module in its `test` block for nested tests to run. `zig build test` produces no output on success.

**C library builds:** `buildFreetype()` and `buildHarfbuzz()` in `build.zig` return `*std.Build.Step.Compile`. `@cImport` wrappers need `mod.addIncludePath()` — `linkLibrary` alone does not propagate headers.

**Vulkan binding generation:** Manual pattern — get the `vulkan-zig-generator` artifact, run it with `registry/vk.xml`, create a module from the output:
```zig
const registry = b.dependency("vulkan_headers", .{}).path("registry/vk.xml");
const vk_gen = b.dependency("vulkan", .{}).artifact("vulkan-zig-generator");
const vk_generate_cmd = b.addRunArtifact(vk_gen);
vk_generate_cmd.addFileArg(registry);
const vulkan_zig = b.addModule("vulkan-zig", .{
    .root_source_file = vk_generate_cmd.addOutputFileArg("vk.zig"),
});
```

**Vulkan filtered dispatch:** `vk.DeviceWrapper(.{...})` does NOT exist. Define a struct with `?vk.PfnXxx = null` fields, pass to `vk.DeviceWrapperWithCustomDispatch(YourStruct)`. See `src/gpu/context.zig` for the `HeavySlugDispatch` pattern.

**Motor math:** `src/math/pga.zig` — storage is `[4]f32` extern struct (GPU ABI). Arithmetic uses `@Vector(4,f32)` + `@splat` + `@shuffle` internally. `Motor.apply` uses `@Vector(2,f32)`.

## Zig 0.16.0-dev Gotchas

- `b.addLibrary(.{ .linkage = .static })` — NOT `b.addStaticLibrary()` (removed in 0.16)
- `@cImport` needs explicit `mod.addIncludePath()` — `linkLibrary` alone doesn't propagate headers
- `zig build test` is silent on success (exit 0, no output)
