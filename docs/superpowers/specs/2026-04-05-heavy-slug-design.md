# heavy-slug Design Spec
**Date**: 2026-04-05  
**Status**: Approved  
**Stack**: Zig 0.16.0-dev.3091 · Slang 2026.5.2 · WebGPU (zgpu/wgpu-native) · HarfBuzz 14.1.0 · FreeType 2.14.3

---

## 1. Problem Statement

Build an industrial-grade GPU text rendering library for embedding in game engines and real-time applications. The caller provides a `wgpu::Device`; the library owns no window or swapchain. Requirements:

- Full Unicode shaping (HarfBuzz, all scripts including CJK)
- Slug algorithm GPU rasterization (exact quadratic Bézier coverage, no bitmap atlas)
- Zero heap allocation in the render loop
- Stateless API surface — caller manages frame lifecycle

---

## 2. Key Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Rendering backend | WebGPU via zgpu (SPIR-V) | Cross-platform; D3D12/DXIL deferred |
| Curve type | Quadratic only | FreeType decomposes CFF cubics; simpler GPU shader |
| HarfBuzz/FreeType integration | Direct C API via `@cImport` | Full control; versions pinned in `build.zig.zon` |
| CPU blob encoding | `libharfbuzz-gpu` (`hb_gpu_draw_encode`) | Battle-tested Slug encoding; avoids reimplementing band decomposition |
| GPU blob format | HarfBuzz RGBA16I texel atlas (`array<vec4<i32>>`) | Directly consumed by ported fragment shader |
| Shader language | Slang (port of `hb-gpu-fragment.wgsl` + PGA vertex) | Generics, Specialization Constants, Reflection |
| Vertex transform | PGA Motor Cl(2,0,1) replacing `mat4x4` | Unified 2D rigid-body transform; compact 4-float encoding |
| Glyph cache | Two-level Hot/Cold | Hot slab for frequent glyphs; Cold ring for rare/new; suitable for CJK scale |
| Rendering mode | GPU-driven indirect | CPU writes `GlyphCommand`; compute pre-pass expands to vertices + culls |

## 3. Architecture Overview

```
┌─────────────────────────────────────────────────────┐
│                    Caller (Game Engine)              │
│  provides: wgpu::Device, wgpu::Queue, wgpu::RenderPass│
└────────────────────┬────────────────────────────────┘
                     │ heavy_slug public API
┌────────────────────▼────────────────────────────────┐
│                   TextRenderer                       │
│  begin() → drawText() × N → flush(pass)             │
└──┬──────────────┬──────────────────┬────────────────┘
   │              │                  │
   ▼              ▼                  ▼
FontContext    GlyphCache         GpuAtlas
(FT_Face +    (Hot Slab +        (wgpu Buffer
 hb_font_t)    Cold Ring)         array<vec4<i32>>)
   │              │
   ▼              ▼
hb_gpu_draw_encode()  →  hb_blob_t  →  upload to atlas slot
```

**Data flow per frame:**

```
CPU:
  TextRenderer.begin()
    └─ reset GlyphCommand write pointer

  TextRenderer.drawText(font, text, motor, color)
    ├─ hb_shape(font, text) → GlyphInfo[] + GlyphPos[]     [cached by text_hash]
    ├─ for each glyph: cache lookup → hot slot or cold slot
    │    miss: hb_gpu_draw_encode() → upload hb_blob_t to atlas
    └─ append GlyphCommand { motor, atlas_offset, em_bounds, color }

  TextRenderer.flush(pass)
    └─ dispatch GlyphExpandCS → DrawIndirect

GPU:
  [Compute] GlyphExpandCS
    ├─ GlyphCommand[] → GlyphVertex[4] per glyph (position + normal)
    └─ frustum cull → DrawIndirectBuffer

  [Render] DrawIndirect
    ├─ Vertex: hb_gpu_dilate (Slang) + Motor → clip-space
    └─ Fragment: hb_gpu_atlas decode + Slug coverage (Slang port of hb-gpu-fragment.wgsl)
```

## 5. Public API

```zig
pub const PipelineVariant = enum { auto, standard, subpixel, fast };

pub const InitOptions = extern struct {
    hot_slab_bytes:       u32 = 8 * 1024 * 1024,
    cold_ring_bytes:      u32 = 2 * 1024 * 1024,
    max_glyphs_per_frame: u32 = 16_384,
    promote_frames:       u8  = 3,
    pipeline_variant:     PipelineVariant = .auto,  // .auto queries wgpu adapter properties
};

pub const FontContext = opaque {};

pub const TextRenderer = struct {
    pub fn init(device: *zgpu.wgpu.Device, queue: *zgpu.wgpu.Queue,
                allocator: std.mem.Allocator, options: InitOptions) !TextRenderer;
    pub fn deinit(self: *TextRenderer) void;

    pub fn loadFont(self: *TextRenderer, path: []const u8, size_px: f32) !*FontContext;
    pub fn unloadFont(self: *TextRenderer, ctx: *FontContext) void;

    // Render loop — zero heap allocation after init
    pub fn begin(self: *TextRenderer) void;
    pub fn drawText(self: *TextRenderer, font: *FontContext, text: []const u8,
                    motor: pga.Motor, color: [4]f32) error{GlyphCapacityExceeded, ShapingFailed}!void;
    pub fn drawStaticText(self: *TextRenderer, font: *FontContext, text: []const u8,
                          motor: pga.Motor, color: [4]f32, cache_key: u64) !void;
    pub fn flush(self: *TextRenderer, pass: *zgpu.wgpu.RenderPassEncoder) void;
};
```

## 6. GPU Data Structures

`src/slug/band.zig` is **auto-generated** by `tools/layout_gen.zig` from `shaders/generated/layout.json`. It is committed to the repo so the build is reproducible without running `slangc`, but must not be edited manually.

Generated output shape (illustrative — actual offsets are Slang-authoritative):

```zig
// AUTO-GENERATED by tools/layout_gen — do not edit manually
// Source: shaders/generated/layout.json  (slangc --reflect slug_text.slang)
const std = @import("std");

pub const GlyphVertex = extern struct {
    position: [2]f32,       // offset  0
    texcoord: [2]f32,       // offset  8
    normal:   [2]f32,       // offset 16
    jac:      [4]f32,       // offset 24
};
comptime {
    std.debug.assert(@sizeOf(GlyphVertex)             == 40);
    std.debug.assert(@offsetOf(GlyphVertex, "jac")    == 24);
}

pub const GlyphInstance = extern struct {
    motor:        [4]f32,   // offset  0
    color:        [4]f32,   // offset 16
    atlas_offset: u32,      // offset 32
    _pad_0:       [12]u8,   // offset 36  ← inserted by layout_gen, not hand-written
};
comptime {
    std.debug.assert(@sizeOf(GlyphInstance)                    == 48);
    std.debug.assert(@offsetOf(GlyphInstance, "atlas_offset")  == 32);
}

pub const GlyphCommand = extern struct {
    motor:        [4]f32,   // offset  0
    color:        [4]f32,   // offset 16
    atlas_offset: u32,      // offset 32
    em_x_min:     f32,      // offset 36
    em_y_min:     f32,      // offset 40
    em_x_max:     f32,      // offset 44
    em_y_max:     f32,      // offset 48
    _pad_0:       [4]u8,    // offset 52  ← inserted by layout_gen
};
comptime {
    std.debug.assert(@sizeOf(GlyphCommand) == 56);
}
```

`layout_gen` algorithm:
1. Parse JSON array of struct descriptors: `[{ name, fields: [{ name, type, offset, size }] }]`
2. Sort fields by offset; scan gaps between `field[i].offset + field[i].size` and `field[i+1].offset`
3. Insert `_pad_N: [gap]u8` for each gap
4. Emit `comptime` assert for `@sizeOf` (total size from JSON) and `@offsetOf` for every non-pad field
5. Exit code 1 if any field has `size % 4 != 0` or struct total size is not 16-byte aligned (WebGPU uniform requirement)

## 7. Slang Shader Design

### 7.1 pga.slang

```slang
struct Motor {
    float4 m; // [s, e12, e01, e02]

    float2 apply(float2 p);          // sandwich product
    float4x4 toMat(float4x4 proj);   // expand to clip-space matrix for hb_gpu_dilate
    static Motor fromTranslation(float2 t);
    static Motor fromRotation(float angle);
    static Motor compose(Motor a, Motor b);
}
```

### 7.2 slug_text.slang

Vertex stage: exact port of `hb_gpu_dilate` (WGSL → Slang), `mat4x4f` replaced by `Motor.toMat(proj)`.

Fragment stage: exact port of `hb-gpu-fragment.wgsl` — `hb_gpu_decode_glyph` + `hb_gpu_calc_coverage`. No logic changes; binding names updated to Slang convention.

Specialization constants:

```slang
[SpecializationConstant] static const bool ENABLE_SUBPIXEL   = false;
[SpecializationConstant] static const bool ENABLE_MSAA_SMALL = true;
```

Subpixel path (LCD): compute coverage at `texcoord ± ddx_fine(texcoord.x)/3` for R and B channels.

### 7.3 glyph_expand.slang

Compute shader (workgroup 64): reads `GlyphCommand[]`, writes `GlyphVertex[4]` per glyph (corners from `em_bounds`, normals = normalized diagonal directions), frustum-culls, writes `DrawIndirectBuffer` via atomic.

## 8. Build System

### 8.1 build.zig.zon dependencies

```zig
.dependencies = .{
    .zmath        = .{ .url = "...", .hash = "..." },
    .zgpu         = .{ .url = "...", .hash = "..." },
    // hashes populated via: zig fetch --save <url>
    .freetype_src = .{ .url = "https://nongnu.askapache.com/freetype/freetype-2.14.3.tar.gz",
                       .hash = "<zig fetch --save to populate>" },
    .harfbuzz_src = .{ .url = "https://github.com/harfbuzz/harfbuzz/releases/download/14.1.0/harfbuzz-14.1.0.tar.xz",
                       .hash = "<zig fetch --save to populate>" },
},
```

### 8.2 C library compilation

`buildFreetype()` and `buildHarfbuzz()` are standalone functions in `build.zig` returning `*Build.Step.Compile`. HarfBuzz is configured with `-DHAVE_FREETYPE=1` and links the FreeType artifact; HarfBuzz's own FreeType subpackage is disabled. Both compile from upstream tarballs as static libraries.

HarfBuzz is built with `-DHAVE_HB_GPU=1` to include `libharfbuzz-gpu` sources (`hb-gpu-draw.cc`, `hb-gpu-shaders.cc`, `hb-gpu-cu2qu.hh`).

### 8.3 Build Pipeline: Slang → Reflect → Codegen → Compile

Three sequential stages before any Zig compilation:

**Stage 1 — SlangReflectStep** (new): invokes `slangc --reflect` on `slug_text.slang` and `glyph_expand.slang`, emitting `shaders/generated/layout.json`. Contains per-struct field names, types, byte offsets, and sizes as reported by the Slang compiler.

**Stage 2 — LayoutGenStep**: builds and runs `tools/layout_gen.zig` as a host executable. Reads `layout.json`, walks each buffer type, and writes `src/slug/band.zig`:
- Inserts `_pad_N: [M]u8` fields automatically at any gap between reported offsets
- Appends `comptime` assertions for `@sizeOf` and `@offsetOf` of every field
- Emits `// AUTO-GENERATED — do not edit manually` header
- Fails the build (non-zero exit) if any Slang type uses unsupported alignment or a non-power-of-two size

**Stage 3 — SlangCompileStep**: compiles three pipeline variants to SPIR-V (`slug_standard.spv`, `slug_subpixel.spv`, `slug_fast.spv`). SPIR-V embedded via `@embedFile` in `pipeline.zig`.

Build order:
```
SlangReflectStep
  └─ LayoutGenStep (produces src/slug/band.zig)
       └─ SlangCompileStep (produces *.spv)
            └─ zig build lib (compiles Zig, links ft + hb + zmath + zgpu)
```

`tools/layout_gen.zig` is a standalone executable with no dependencies beyond `std`; it only runs on the build host and is never linked into the library.

## 9. Memory Strategy

| Scope | Allocator | Lifetime |
|---|---|---|
| Font load + FreeType outline decompose | `ArenaAllocator` | freed at end of `loadFont` |
| GlyphCache entries | `MemoryPool(GlyphCacheEntry)` | lives with `TextRenderer` |
| Hot Slab GPU Buffer | pre-allocated slab | lives with `TextRenderer` |
| Cold Ring GPU Buffer | pre-allocated ring | lives with `TextRenderer` |
| GlyphCommand write buffer | pre-allocated, reset in `begin()` | per-frame |
| GlyphVertex/Instance GPU buffers | compute-written, no CPU alloc | per-frame |
| HarfBuzz shaping results | `ArenaAllocator` (frame-scoped) | reset in `flush()` |
| Static text texture cache | pool of `wgpu.Texture` | lives with `TextRenderer` |

Render loop invariant: **no `allocator.alloc` calls between `begin()` and `flush()`**.

## 10. Two-Level Glyph Cache (Hot/Cold)

**Hot Slab** (`hot_slab_bytes`, default 8 MB):
- Slab allocator over a persistent `wgpu.Buffer`
- Populated at `loadFont` for ASCII (U+0020–U+007E) + promoted glyphs
- Evicted only on `unloadFont`

**Cold Ring** (`cold_ring_bytes`, default 2 MB):
- Ring allocator over a persistent `wgpu.Buffer`
- New/rare glyphs land here on first encounter
- Promotion: glyph seen in `promote_frames` (default 3) consecutive frames → copied to Hot Slab, Cold slot recycled

**Cache key**: `(font_id: u32, glyph_id: u32, size_bucket: u8)` — size quantized to 8 buckets (8–256 px) to limit unique blobs per glyph.

**Shaping cache**: `(text_hash: u64, font_id: u32, features_hash: u32)` → `[]hb_glyph_info_t + []hb_glyph_position_t` stored in a fixed-capacity LRU (default 256 entries). Avoids re-shaping identical strings.

## 11. Pipeline Variants

| SPIR-V | `ENABLE_SUBPIXEL` | `ENABLE_MSAA_SMALL` | Use case |
|---|---|---|---|
| `slug_standard.spv` | false | true | OLED, HDR, non-LCD |
| `slug_subpixel.spv` | true | true | LCD displays |
| `slug_fast.spv` | false | false | Game HUD, high frame rate |

Runtime selection at `TextRenderer.init` via `InitOptions.pipeline_variant` (default: auto-detect from display).

## 12. wgpu Bind Group Layout

```
Group 0 — per-frame globals
  b0: uniform  GlobalUniforms { proj: mat4x4f, viewport: vec2f, _pad: vec2f }
  b1: storage  hb_gpu_atlas[] read-only   // array<vec4<i32>>

Group 1 — per-font
  b0: uniform  FontUniforms { em_scale: f32, ascender: f32, descender: f32, _pad: f32 }

Vertex buffer slot 0: GlyphVertex[]    step_mode=vertex   (compute-written)
Vertex buffer slot 1: GlyphInstance[]  step_mode=instance (compute-written)
```

## 13. Out of Scope (v1)

- D3D12 / DXIL backend
- Cubic Bézier native support
- Variable font axis animation
- Async background glyph encoding (deferred to v2)
- Right-to-left UI layout (HarfBuzz shapes correctly; layout engine is caller's responsibility)
